# BOM 对比工具 - 功能需求 V2

## 概述

嘉立创 EDA 扩展，用于对比两个 BOM（物料清单）文件的差异，以表格形式直观展示变更内容。支持多种文件格式导入，提供高效的差异分析和可视化能力。

---

## 界面布局

### 整体结构

- 左右双面板布局，左侧为"旧文件"，右侧为"新文件"
- 顶部为文件操作栏
- 中间为 BOM 数据表格（支持同步滚动）
- 底部为结果汇总栏和操作按钮
- 中间分隔栏可拖拽调整左右面板宽度

### 顶部文件操作栏

| 元素 | 说明 |
|------|------|
| 文件路径输入框 | 显示当前加载的文件路径 |
| 浏览按钮（...） | 打开文件选择对话框 |
| 清空按钮 | 清除当前加载的文件 |
| 另存为按钮 | 将当前 BOM 数据另存为文件 |
| 关于嘉立创按钮 | 显示关于信息（仅右侧） |

### BOM 数据表格

#### 列定义

| 列名 | 说明 |
|------|------|
| 位号 (Designator) | 元器件位号，作为对比基准 |
| 封装 (Footprint) | 元器件封装类型 |
| 数量 (Quantity) | 元器件数量 |
| 制造商 (Manufacturer) | 制造商名称 |
| LCSC Part Number | 嘉立创商城物料编号 |
| Value | 元器件参数值 |
| Description | 元器件描述 |

#### 每行操作

- 详情按钮：查看该行元器件的详细对比信息（弹窗展示新旧值逐字段对比）

## 核心功能

### 1. 文件导入

#### 支持格式

| 格式 | 说明 |
|------|------|
| CSV (.csv) | 逗号分隔值文件 |
| TXT (.txt) | 文本文件（支持制表符、逗号、分号等常见分隔符自动识别） |
| XLS (.xls) | Excel 97-2003 格式 |
| XLSX (.xlsx) | Excel 2007+ 格式 |

#### 导入方式

| 方式 | 说明 |
|------|------|
| 文件选择对话框 | 点击浏览按钮，弹出系统文件选择器 |
| 拖拽导入 | 将文件直接拖拽到对应面板区域即可加载 |

#### 拖拽交互细节

- 拖拽文件悬停在面板上方时，显示可放置区域的视觉提示（虚线边框 + 半透明遮罩 + 提示文字）
- 支持拖拽到左侧面板（旧文件）或右侧面板（新文件）
- 拖拽非支持格式的文件时，显示格式不支持的提示
- 同时拖入两个文件时，按拖入顺序分别加载到左右面板

#### 文件解析

- 自动识别文件编码（UTF-8、GBK、GB2312 等）
- Excel 文件支持选择工作表（Sheet），默认加载第一个 Sheet
- 自动识别表头行（首行或包含 Designator/位号 关键字的行）
- 支持列名智能映射：自动匹配常见的中英文列名变体（如"位号"↔"Designator"，"封装"↔"Footprint"）

### 2. 列映射配置

- 文件加载后，若列名无法自动匹配，弹出列映射配置面板
- 用户可手动指定源文件列与标准列的对应关系
- 记住用户的映射配置，下次加载相同格式文件时自动应用

### 3. BOM 对比

- 以**位号（Designator）**作为对比基准
- 用户可自定义选择参与对比的列（默认：封装、数量、制造商、LCSC Part Number、Value）
- 点击"执行BOM比对"按钮触发对比
- 两侧文件均加载完成后，支持自动执行对比（可在设置中开关）

### 4. 差异高亮显示

| 颜色 | 含义 |
|------|------|
| 蓝色/青色单元格 | 该单元格的值在新旧文件中不同（标记在有变化的一侧） |
| 橙色/黄色整行 | 新文件中多出的元器件（旧文件中不存在） |
| 红色/粉色整行 | 旧文件中存在但新文件中缺失的元器件 |
| 绿色单元格 | 有差异的具体字段高亮 |
| 无高亮 | 该行/单元格在两个文件中完全一致 |

### 5. 结果汇总

底部状态栏显示对比结果统计：

- 完全相同的行数
- 有差异的行数
- 新文件多出的行数
- 新文件缺失的行数（旧文件有但新文件没有）

格式示例：
> 结果汇总：以位号为对比基准，共对比了【封装】【数量】【位号】【制造商】列，新文件相对旧文件，0行完全相同，12行有差异，多出2行，缺失1行

### 6. 导出功能

- 另存为：支持导出为 CSV / XLSX 格式
- 导出对比报告：将差异结果导出为独立文件，包含变更摘要和详细差异

## 对比逻辑

1. 以位号为主键匹配左右两侧的行
2. 匹配成功的行，逐列对比用户选定字段的值
3. 值不同的单元格进行高亮标记
4. 右侧多出的行（左侧无对应位号）标记为"新增"，归类到右侧表格底部
5. 左侧有但右侧无对应位号的行标记为"缺失"，归类到右侧表格底部（灰色）
6. 对比结果支持按差异类型筛选和导航

## 技术约束

- 作为嘉立创 EDA 扩展运行，需遵循扩展 API 规范
- 前端渲染需考虑大数据量场景（虚拟滚动）
- 文件解析在前端完成，无需后端服务
- 需兼容嘉立创 EDA 的主题风格
