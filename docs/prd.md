## 1. 产品背景

### 1.1 业务背景

在电子产品开发过程中，BOM（物料清单）会随着设计迭代频繁变更。工程师需要快速识别两个版本 BOM 之间的差异，以确保物料变更被正确追踪、采购计划及时调整、生产不受影响。

目前工程师通常依赖手动逐行比对或通用 diff 工具，效率低且容易遗漏。

### 1.2 目标用户

- 硬件工程师：设计迭代过程中对比不同版本 BOM
- 采购人员：确认物料变更范围，调整采购计划
- 项目经理：审核 BOM 变更，评估影响范围

### 1.3 产品目标

- 提供专业的 BOM 差异对比能力，集成在嘉立创 EDA 工作流中
- 降低 BOM 对比的操作门槛和时间成本
- 减少人工对比的遗漏风险

### 1.4 成功指标

| 指标 | 目标 |
|------|------|
| 对比操作耗时 | 从加载文件到获得结果 < 5 秒（1000 行以内） |
| 差异识别准确率 | 100%（基于位号匹配的逻辑对比） |
| 支持文件规模 | 单文件 10000+ 行流畅运行 |

## 3. 详细功能需求

### 3.1 文件导入

#### 3.1.1 支持格式

| ID | 格式 | 扩展名 | 解析方式 |
|----|------|--------|----------|
| F-1.1 | CSV | .csv | 逗号分隔，支持引号包裹字段 |
| F-1.2 | TXT | .txt | 自动识别分隔符（制表符 > 逗号 > 分号） |
| F-1.3 | XLS | .xls | Excel 97-2003 二进制格式 |
| F-1.4 | XLSX | .xlsx | Excel 2007+ OpenXML 格式 |

#### 3.1.2 导入方式

| ID | 方式 | 触发条件 | 行为 |
|----|------|----------|------|
| F-1.5 | 文件选择器 | 点击浏览按钮 | 弹出系统文件对话框，过滤显示支持的格式 |
| F-1.6 | 拖拽导入 | 拖拽文件到面板区域 | 解析并加载文件内容到对应面板 |

#### 3.1.3 拖拽交互规则

| ID | 场景 | 行为 |
|----|------|------|
| F-1.7 | 文件悬停在面板上方 | 显示放置区域提示：虚线边框 + 半透明蓝色遮罩 + "释放以加载文件"文字 |
| F-1.8 | 文件格式不支持 | 遮罩变为红色 + 显示"不支持该文件格式" |
| F-1.9 | 释放文件 | 开始解析，显示加载进度 |
| F-1.10 | 同时拖入两个文件 | 第一个文件加载到左侧，第二个加载到右侧 |

#### 3.1.4 文件解析规则

| ID | 规则 | 说明 |
|----|------|------|
| F-1.11 | 编码识别 | 自动检测 UTF-8（含 BOM）、GBK、GB2312，默认 UTF-8 |
| F-1.12 | 表头识别 | 扫描前 5 行，找到包含"Designator"或"位号"关键字的行作为表头 |
| F-1.13 | Sheet 选择 | Excel 文件含多个 Sheet 时，弹出选择器，默认选中第一个 |
| F-1.14 | 空行处理 | 跳过全空行，数据从表头下一行开始 |
| F-1.15 | 列名映射 | 自动匹配中英文列名变体（见映射表） |

#### 3.1.5 列名映射表

| 标准列名 | 可识别的变体 |
|----------|-------------|
| Designator | 位号, Ref, Reference, RefDes |
| Footprint | 封装, Package, Pkg |
| Quantity | 数量, Qty, Count |
| Manufacturer | 制造商, MFR, Mfg |
| LCSC Part Number | LCSC, 立创商城编号, LCSC Part, JLC Part |
| Value | 值, Val, 参数 |
| Description | 描述, Desc, 说明 |

|----|------|------|

| F-2.5 | 重置映射 | 支持一键恢复自动映射结果 |

### 3.3 BOM 对比

#### 3.3.1 对比配置

| ID | 功能 | 说明 |
|----|------|------|
| F-3.1 | 对比基准 | 固定以位号（Designator）为主键进行行匹配 |
| F-3.2 | 对比列选择 | 用户可勾选参与对比的列，默认全选 |
| F-3.3 | 触发方式 | 点击"执行BOM比对"按钮 |
| F-3.4 | 自动对比 | 可选：两侧文件加载完成后自动触发（设置项，默认关闭） |

#### 3.3.2 对比算法

| ID | 步骤 | 说明 |
|----|------|------|
| F-3.5 | 行匹配 | 以位号为 key，建立旧文件行 → 新文件行的映射 |
| F-3.6 | 字段对比 | 对匹配成功的行，逐列比较选定字段的值（忽略前后空格） |
| F-3.7 | 新增识别 | 新文件中存在但旧文件中无对应位号的行，标记为"新增" |
| F-3.8 | 缺失识别 | 旧文件中存在但新文件中无对应位号的行，标记为"缺失" |
| F-3.9 | 重复位号 | 若同一文件中存在重位号，按出现顺序逐一匹配 |

## 5. 非功能需求

### 5.1 性能要求

| 场景 | 要求 |
|------|------|
| 文件加载（<1000行） | < 1 秒 |
| 文件加载（1000-10000行） | < 3 秒 |
| 对比计算（<1000行） | < 500 毫秒 |
| 对比计算（1000-10000行） | < 2 秒 |
| 表格滚动帧率 | ≥ 30 FPS |

### 5.2 兼容性

| 项目 | 要求 |
|------|------|
| 嘉立创 EDA 版本 | 兼容当前最新版本 |
| 操作系统 | Windows 10+、macOS 10.15+ |
| 文件编码 | UTF-8、GBK、GB2312、UTF-16 |
| BOM 规模 | 支持 10000+ 行 |

### 5.3 可用性

- 首次使用无需配置即可完成基本对比流程
- 关键操作（加载、对比、导航）不超过 3 次点击
- 错误信息使用用户可理解的语言，提供解决建议

## 6. 异常处理

| 场景 | 处理方式 |
|------|----------|
| 文件格式不支持 | Toast 提示"不支持该文件格式，请使用 CSV/TXT/XLS/XLSX 文件" |
| 文件解析失败 | 弹窗提示具体错误原因，提供"重试"和"选择其他文件"选项 |
| 未找到表头 | 提示"未识别到 BOM 表头，请确认文件格式"，提供手动指定表头行的选项 |
| 位号列缺失 | 提示"未找到位号列，请在列映射中指定" |
| 文件过大（>50MB） | 提示"文件过大，建议拆分后再对比" |
| 仅加载一侧文件时点击对比 | 提示"请先加载两侧 BOM 文件" |
